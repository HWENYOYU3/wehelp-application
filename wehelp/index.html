【求學經歷】
我叫蔡嘉文，去年畢業於臺北大學企業管理學系。大學主修財務工程，除了學習財務理論，也對基本的C++、Excel VBA有所認識；研究所延續所學，針對臺灣公司進行財務研究。
【關於我】
自身熱愛有挑戰性的事物，喜歡獨立解決問題。在面對不熟悉的事物時，我不會希望直接得到答案，因為透過自己摸索所獲得成果會更令我感到印象深刻。在撰寫程式的過程中發現，能夠透過所寫的語句來創造一個實際看得到的東西會讓我感到滿滿的成就感，因此決定投入轉職。

大學時期，因為就讀科系屬於半個工程系，曾碰過一些程式語言(Excel VBA、C++)。當時，我因為搞不懂迴圈是什麼，不理解為什麼程式邏輯是這樣的運作，就未深入鑽研，而改為學習金融相關的知識。

不過隨著時間，發現自己雖然不排斥金融知識，但不排斥和喜歡終究是有差距的。在研究所期間，因為論文需求而重拾程式語言的過程中，我發現在面對不確定的事物時，我會主動去搜尋辦法、查閱相關資料並設法解決，從中我感受到不同於學習金融知識時的踏實感，當時便萌生了撰寫程式的想法，但畢竟科系唸了、證照考了，貿然的「轉職」可能只是一時的衝動。

真正讓我下定決心的關鍵是在當兵期間，雖然當時仍在準備一些瑣碎的證照，但我同時也在搜尋金融業內有什麼工作會牽涉到程式語言。而在學習資料庫的途中，看到了這個訓練計畫，發現這個計畫涉及了前端、後端以及資料庫，屬於一個完整的任務訓練，結合之前的想法，讓我在退伍後決定不再繼續考金融證照，而是全心投入成為一個軟體工程師。

準備申請書的這段期間，我主要透過網路上的資源(彭彭老師的YT、Udemy)以及線下書籍《網頁設計完全攻略》，學習了HTML、CSS的基礎原理。在不懂的地方，我會去查詢其他人撰寫的範例，理解後在另開一個小檔案試做，確定沒問題後便使用在自己的申請書上，展示自己的所學。

同時，我也學習了JavaScript，由於此份申請書的需求，雖然我未將JS融入在此份申請書上，但我也另外製作了一個多數學習程式語言的人都會練習的貪食蛇遊戲，來表現我學習的成果。 這個遊戲使用了我這段時間學習到的JS內容，包括基礎語法、瀏覽器儲存庫、DOM操作等，我也在製作過程中通過不斷思考，慢慢地將一個一個功能寫出來，最後完成一段相對完整的程式碼。

1月
一月迄今

下定決心磨練自己的程式技能，在上網學習相關技能時發現了 WeHelp 計畫，決定全心投入，並在此期間學習相關技能，準備申請書。

學習時間的規劃上，我預計一天約投入8小時，一週至少50小時來進行學習。

平日，我會把這個訓練視作正常的上班時間，主要安排白天進行學習，全心投入訓練，目的在於即使時間相對彈性，但仍須保持高度自律，不能因為彈性就破壞了生活作息；而晚上時段則給予適當的彈性，視當週進度進行安排。

假日，由於羽球是我的興趣，也是我紓解壓力的一項重要運動，因此我每個禮拜天仍會安排下午時段去上課，維持運動習慣並放鬆； 其餘時間則適情況調整，一天至少安排4-6小時來進行學習。

我目前還沒有明確的想法來決定我想要去的公司，原因是目前的我在瀏覽求職網站時，發現有太多重複出現，但我仍未接觸過、甚至聽都沒聽過的技術，代表多數公司所要求軟體工程師所必備的技能我還尚未掌握， 因此在決定未來想進入的公司之前，我想先把基礎磨練好，到時候才有選擇公司的本錢(好消息是，隨著在準備申請的這段時間，我逐漸看得懂那些網頁開發前端相關的技術有哪些了！)。

最近一次的負面情緒，是在我就讀研究所時，家人因為意外住院而發生的。

當時正屬論文的繁重時期，加上和朋友因為一些不愉快正在冷戰，接著，家人爬山時意外失足而住院，將我的精神逼到了非常緊繃的地步。 我感覺到思緒非常的混雜，心裡的想法會互相糾結，「今天晚上要去醫院」、「但兩天後要完成論文進度的還差一點沒完成」、「下禮拜又要看到那位朋友」等每個事件的想法互相糾纏，形成了一股負面循環。

而我在當時處理這個事情的方法是，先冷靜下來，然後將這些混雜在一起的事情一件一件切割開來處理，再依照重要程度進行排序。

最重要的家人排在第一，在家人未出院前以他們為重；而對於論文進度的擔憂則可以後續花時間補回；而與朋友間的矛盾則可以在家人的狀況穩定之後再想辦法解開誤會。

有了整理好的想法後，我心中的不安就不再擴大，能夠專注地處理一件事而不再陷入泥沼之中。

從上面的經歷讓我明白，有時候很多事情在同一時間向我襲捲而來時，心裡會冒出許多想法，導致思緒無法清晰，這些無謂的顧慮累積起來就會形成一股負面壓力。這時候就需要給自己一個時間，好好冷靜下來，重整一下思緒，將問題分開看待，按照優先緩急進行排序。當靜下心想好辦法之後，我就不會再陷入這股負面情緒，而是能專注於解決問題。

開發這份網頁時，我發現在我完成網頁的基本架構和樣式後，會逐漸開始想要加入一些有趣的功能。 雖然這種突然冒出的想法在當下不一定能想到做法去將它實現，但是透過搜尋資料、不斷試錯與更改，我還是能把它做出來。而在最後成功將功能放在網頁上的那一刻，帶給了我滿滿的成就感，也促使我會想要再學習更多有趣的功能。

舉例一個功能——漢堡選單。一開始的想法其實是做一個可以折疊的訊息，上網一查，發現透過 HTML 內建的summary tag 就可以輕易完成，但其運行結果卻不能進行細節的微調，達到我所想要的效果，便轉而製作能客製化的漢堡選單。

製作過程中，一開始要如何畫出線條就苦惱了我許久，在搜尋了相關範例後，我成功的用 div 填滿其 background-color 的方式畫出線條，並讓它進行 transform 位移；而在後續，我還遇到如何讓未顯示的元素不佔據位置、transition 無法套用設定等許多問題待我一一去解決。

而在處理問題時，我發現，即使先在空白的檔案內做出預期的功能的漢堡選單，但真正到了要將它放入目前的專案時，往往會因為已經做好架構的網頁會有許多層級的關係等因素，而不小心在錯誤的位置進行 CSS 設定，導致結果出錯，此時就需要耐心地去檢查，找出問題。這個過程也讓我明白撰寫程式需要更細心、更小心地去處理每一行的程式，以及釐清其背後的邏輯，將指令放在正確的位置，才能避免出錯。 而在這些問題依依解決，成功將漢堡選單放在我所希望的位置並達到心中預期的功能的時，除了感到開心、收穫滿滿成就之外，也知道自己又進步了一點。

我認為人與人之間的交往原則為「不交惡」。 由於無法保證自己在當下遇到的人，會在未來扮演什麼樣的角色，因此我在面對初次見面的人時皆秉持著互相尊重、包容的態度，但仍會保持著一些距離；至於能否深交，則取決於日後的互動中能否信任、幫助以及理解彼此的價值觀。

感謝彭彭老師在網路上無私分享的資源，以深入淺出的方式來讓想要學習程式的人以最簡單的方式獲得最直觀的結果，進而讓觀看者產生興趣。

兩個月前，我連 HTML 是什麼都還不知道，而現在，我因為 YouTube 的免費教學產生了興趣，進而去尋找許多更細節、更完整的教學來完成這份網頁。過去的我很害怕迴圈，看到層層的結構就會下意識地逃避它；而現在，雖然我仍無法快速的掌握它，但對於其基本的原理的了解相較於過去的我有著很顯著的成長，更重要的是，我不再會因為看到落落長的程式碼而感到心生恐懼，而是會耐下性子一步一步去頗析程式碼，明白其運作的邏輯。

這些是我在準備申請階段時的就獲得的巨大收穫，希望後續能成功加入 WeHelp ，成為厲害學長姐的一員。最後，感謝 WeHelp 團隊播冗閱讀我的申請書，祝福平安順心。
